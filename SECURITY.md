# Security Policy

## Scope

This is an educational portfolio project. It is not a production intrusion-detection or automated-response system.

## Reporting

Please open a GitHub issue for non-sensitive defects. Do not post credentials, private data, exploit payloads, or active secrets in an issue.

## Credential handling

- Never commit AWS access keys, session tokens, `.env` files, or copied learner-lab credentials.
- Prefer IAM roles or AWS IAM Identity Center.
- Keep S3 public access blocked.
- Restrict IAM policies to the specific bucket, database, workgroup, and log group used by this project.

## Model-file warning

Python pickle and joblib files can execute code during deserialization. The application should only load artifacts generated by you or supplied through a trusted and verified process.

## Data handling

Use synthetic, public, or explicitly authorized network data. Do not upload campus, employer, customer, or personal network telemetry without written authorization.
